# Contributor Guide

## Purpose

Ivy is designed for multiple collaborators.

This guide defines how to contribute consistently.

---

# Principles

## 1. Be structured
Follow the snapshot schema exactly.

## 2. Be concise
Remove fluff. Keep signal.

## 3. Capture decisions
Not just ideas — what was decided.

## 4. Extract reuse
If something is reusable, make it an artifact.

## 5. Link, don’t duplicate
Prefer references over repetition.

---

# Writing a snapshot

Every snapshot must include:

- Snapshot précis
- Prompt / Trigger
- Context
- Key ideas
- Decisions
- Reusable patterns
- Artifacts created
- Open questions
- Follow-up actions

---

# Style guidelines

## Good
- clear bullets
- explicit decisions
- concrete language

## Bad
- vague summaries
- long paragraphs
- transcript-like text

---

# Metadata rules

- Use valid enums
- Use kebab-case tags
- Do not invent new types casually
- Always set visibility

---

# Common mistakes

## Dumping transcripts
Do not paste raw conversations.

## Missing decisions
Always state what was concluded.

## Over-tagging
Keep tags minimal and meaningful.

## Creating duplicate concepts
Check before creating new ones.

---

# Collaboration model

You are not just writing notes.

You are contributing to a shared system of reasoning.

Make it:
- understandable
- reusable
- extendable

---

# Review checklist

Before committing:

- structure is complete
- metadata is valid
- decisions are explicit
- reusable ideas extracted
- visibility correct
